# AURUM Certification Program

Three-tier practitioner development path aligned to the five AURUM stages.

## Tier 01 — Foundation
**Audience:** Data analysts, junior stewards, business stakeholders
**Duration:** 8 hours self-paced

Modules:
1. What is MDM and why does it matter?
2. The five AURUM stages — what happens where
3. The 7 domains — mastering challenges per domain
4. Data quality fundamentals — completeness, consistency, conformity, accuracy, uniqueness, timeliness
5. The golden record — what it is, how survivorship works

## Tier 02 — Practitioner
**Audience:** MDM implementers, data engineers, stewards
**Duration:** 24 hours with labs

Modules:
1. Running ASSAY — schema inspection and source profiling
2. Running UNEARTH — DQ rule execution and anomaly detection
3. Running REFINE — blocking, matching, survivorship in practice
4. Running UNFURL — publication patterns and consumer management
5. Running MARK — lineage tracking and reverse integration

Labs (in `certification/labs/`):
- Lab 01: Profile the dirty customer dataset
- Lab 02: Write a DQ rule for the product domain
- Lab 03: Score a candidate pair manually, then with AURUM
- Lab 04: Build a golden record from a 4-record cluster
- Lab 05: Trace the lineage of a golden record end-to-end

## Tier 03 — Architect
**Audience:** Data architects, MDM programme leads
**Duration:** 16 hours with design exercises

Modules:
1. Operating model design — roles, responsibilities, governance structure
2. AI augmentation patterns — where ML adds value, where it adds risk
3. Power Platform integration — Dataverse as MDM hub, Power Automate as stewardship engine
4. Multi-domain MDM — cross-domain relationships and reference data
5. Migration patterns — CHARON: assess, map, validate, cutover
