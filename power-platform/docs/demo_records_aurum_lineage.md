## How this maps to the AURUM public-repo reference architecture

All confidence values above are computed by **`refine.matching.matcher.score_pair`** in the public AURUM repo (commit `78223b64c2bb`). The script imports the function directly — see the top of `scripts/load_sample_data.py`. If the AURUM matcher signature changes, the import breaks loudly (no silent drift via a vendored copy).

**The matcher's composite-score formula has FOUR mechanisms, not three:**

1. **Component scores** (each in [0.0, 1.0]):  
   - `name_score` = 0.5 × token-sort-ratio (RapidFuzz) + 0.5 × Jaro-Winkler (jellyfish), both lowercased  
   - `email_score` = `1.0` if exact (post-lowercase + strip), else fuzzy × 0.3 (capped at 0.3)  
   - `phone_score` = `1.0` if last-9-digits identical, else `0.0` (binary — no fuzzy)
2. **Weighted total:** `name_score × 0.65 + email_score × 0.25 + phone_score × 0.10`
3. **NAME-BOOST FLOOR** (the load-bearing fourth mechanism, often missed):  
   `floor = name_score × 0.85` if `name_score >= 0.90` else `0.0`  
   `composite = max(weighted, floor)`  
   The boost recognises a real-world MDM truth: when two records share a near-identical name, that alone is significant evidence of identity even when contact details diverge across systems. Visible above in pairs where the per-record breakdown shows `name_boost_floor` ACTIVE.
4. **Match threshold:** `MATCH_THRESHOLD = 0.65` — `composite >= 0.65` is `is_match=True` (auto-approve to canonical). Below this, steward-review band. Below the candidate-filter floor of `0.55`, not surfaced as a candidate at all.

**Other AURUM patterns demonstrated by the data generation:**

- **Linked-tuple survivorship for geography** (Aisha): `address_line1` + `city` + `country` + `postal_code` are emitted as a single tuple per source. **CRITICAL:** the matcher does NOT score addresses at all — they're a pure UNFURL/survivorship concern, deliberately separated from REFINE matching. Aisha's address divergence is invisible to REFINE; it surfaces only when UNFURL picks one source's whole tuple.
- **Trust score** (every canonical): `0.6 × completeness + 0.4 × diversity`, where completeness is the fraction of 6 key fields populated and diversity = `min(distinct_sources / 3, 1.0)`. Filler canonicals compute trust from these formulas at generation time.
- **`is_golden` flag**: true when trust ≥ 0.6, per the public AURUM convention.
- **Single-Source Promotion** (LOYALTY-specific match_method value 6): demonstrates the AURUM pattern where loyalty-only members can be promoted to canonical without a second-source confirmation, used by the LOYALTY prospect record in pattern #5.
